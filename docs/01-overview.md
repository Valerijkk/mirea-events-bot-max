# 1. Обзор продукта

## Что это

**mirea-events-bot** — чат-бот для мессенджера **МАКС**, автоматизирующий запись
**абитуриентов и студентов РТУ МИРЭА** на мероприятия университета (дни открытых
дверей, мастер-классы, олимпиады, экскурсии, консультации).

Проект разрабатывается для хакатона **«Весенний код» РТУ МИРЭА × VK ×
1С × МКСКОМ**, трек **«МАКС»**, **кейс №2 — «Запись абитуриента на
мероприятие университета»**.

## Что решает

| Боль РТУ МИРЭА                                                        | Как решает mirea-events-bot                                                       |
|-----------------------------------------------------------------------|-----------------------------------------------------------------------------|
| Запись через Google Forms — долго, неудобно, нет интеграции.          | Запись «в 4 тапа» прямо в МАКС: QR на афише → карточка → «Записаться».      |
| Рассылка переноса / отмены вручную в WhatsApp / Telegram.             | Управляемая рассылка по сегментам (только подтверждённые / waitlist / все). |
| Свободное место после отмены пропадает.                               | Автоматическое продвижение из листа ожидания с моментальным уведомлением.   |
| Учёт явки на входе — на бумаге.                                       | QR-сканер: одно нажатие — `attended`, имя+мероприятие на экране.            |
| Никаких напоминаний → высокий no-show rate.                           | Авто-напоминания за 24 часа и за 1 час до начала.                           |
| Нет аналитики: «куда вкладываться, что популярнее».                   | Воронка: записались → подтвердили → пришли. И заполняемость, и явка.        |

## Кто пользуется

1. **Абитуриент / студент МИРЭА** — переходит по deeplink из афиши, записывается,
   получает QR-пропуск, напоминания, может отменить.
2. **Организатор от РТУ МИРЭА** (приёмная комиссия, факультет, студсовет, кафедра) —
   заходит в веб-админку, создаёт мероприятие, публикует, мониторит запись,
   делает рассылки, сканирует QR на входе.
3. **Внешние системы** — могут пользоваться REST API (JWT-авторизация) для
   синхронизации с информационной системой вуза или мобильным сканером.

## Ключевые цифры

* **4 тапа** от QR на афише до подтверждения записи.
* **< 2 секунды** — время от записи до получения QR-кода.
* **2 уровня** уведомлений — за 24 часа и за 1 час.
* **5 сегментов** для рассылок: all, confirmed, waitlist, attended, no_show.

## Что входит в поставку

* Бот в МАКС (long polling в dev, webhook в проде).
* REST API под `/api/v1/...` со Swagger UI.
* React SPA для организаторов (Vite + React 18 + TypeScript + Tailwind).
* Планировщик автоматических напоминаний.
* PostgreSQL — основная БД (docker-compose и локальный dev).

## Что НЕ входит (намеренно)

* SSO с системой вуза — оставлено как точка интеграции через REST API.
* Платёжные сценарии — кейс про запись на бесплатные мероприятия.
* Мобильное приложение сканера — есть веб-страница сканера в админке.
